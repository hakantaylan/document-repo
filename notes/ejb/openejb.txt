@EJB MappedName functionality
https://gist.github.com/dblevins/1650077
"foo"
Implied that foo is a deployment id. Each bean has a unique internal id, this is taken as that ID. The bean is looked up directly from the internal registry via this ID.
"jndi:foo"
Implied that foo is a JNDI name. This name will be looked up via new InitialContext().lookup("foo").
"jndi:ext://bar/foo"
Implied that foo is a JNDI name. This name will be looked up via a lookup on the InitialContext configured in the openejb.xml with the name bar. Essentially internalSuff.getInitialContext("bar").lookup("foo").
this could perhaps be simplified as "jndi:bar:foo"
