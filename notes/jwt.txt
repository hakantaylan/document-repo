1. The only way to prevent the refresh_token to be vulnerable is to never expose it to the client. Even you sign the refresh_token, a hacked browser that steals it will be able to use and abuse. Signing the refresh token is just overwork.

2. Store the refresh_token in the database is a security approach. It makes it easy and fast to destroy a login. Do you know that feature "see a list of devices, and disconnect each of them"? It will be only possible by storing data somewhere.

An alternative would be have an proxy server that manages client session with the access and refresh tokens. By using this approach will never exposes data to client.
